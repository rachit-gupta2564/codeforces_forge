<h3><a href="https://codeforces.com/contest/2195/problem/E" target="_blank" rel="noopener noreferrer">Idiot First Search</a></h3>

<div class="header"><div class="title">E. Idiot First Search</div><div class="time-limit"><div class="property-title">time limit per test</div>2 seconds</div><div class="memory-limit"><div class="property-title">memory limit per test</div>512 megabytes</div><div class="input-file input-standard"><div class="property-title">input</div>standard input</div><div class="output-file output-standard"><div class="property-title">output</div>standard output</div></div><div><p>There is a binary tree of $$$n+1$$$ vertices (<span class="tex-font-style-bf">$$$n$$$ is odd</span>), with vertices labeled $$$0,1,\ldots,n$$$. <span class="tex-font-style-bf">At most one letter</span> can be written on each vertex of the tree, and all vertices initially have nothing written on them. The root of the tree is vertex $$$0$$$.</p><p>In the tree, vertex $$$0$$$ is the parent of vertex $$$1$$$, while all other vertices have either $$$2$$$ children or $$$0$$$ children.</p><p>Bob is lost in one vertex of the tree and wishes to escape the tree by reaching vertex $$$0$$$. This is very easy for most people with common sense. However, since Bob is an idiot, he created a new way of traversing the tree; introducing the "Idiot First Search".</p><p>When Bob is on vertex $$$v$$$ ($$$1 \le v \le n$$$), Bob's movement is determined as follows:</p><ul> <li> If vertex $$$v$$$ is a leaf, Bob <span class="tex-font-style-bf">always</span> moves to the parent of $$$v$$$; otherwise, check the next few conditions. </li><li> If nothing is written on vertex $$$v$$$, Bob writes '<span class="tex-font-style-tt">L</span>' on vertex $$$v$$$ and moves to the left child of $$$v$$$; </li><li> If '<span class="tex-font-style-tt">L</span>' is written on vertex $$$v$$$, Bob <span class="tex-font-style-bf">overwrites it</span> to '<span class="tex-font-style-tt">R</span>' and moves to the right child of $$$v$$$; </li><li> If '<span class="tex-font-style-tt">R</span>' is written on vertex $$$v$$$, Bob <span class="tex-font-style-bf">erases it</span> and moves to the parent of $$$v$$$. </li></ul><p>It takes exactly $$$1$$$ second for Bob to move to an adjacent vertex, so Bob will take exactly $$$x$$$ seconds to perform $$$x$$$ moves.</p><p>It has been shown that regardless of which vertex Bob starts on, Bob can reach vertex $$$0$$$ in a finite (though possibly inexplicably large) amount of time. We don't know who proved it; surely it can't be Bob, but it is definitely proven.</p><p>For each vertex $$$k=1,2,\ldots,n$$$, please determine the total time it takes to reach vertex $$$0$$$ if Bob started on vertex $$$k$$$, in seconds. As the values may be huge, you are only asked to compute them modulo $$$10^9+7$$$.</p></div><div class="input-specification"><div class="section-title">Input</div><p>Each test contains multiple test cases. The first line contains the number of test cases $$$t$$$ ($$$1 \le t \le 10^4$$$). The description of the test cases follows. </p><p>The first line of each test case contains a single integer $$$n$$$ ($$$1 \le n \le 300\,001$$$, <span class="tex-font-style-bf">$$$n$$$ is odd</span>).</p><p>Each of the next $$$n$$$ lines contains two integers $$$l_i$$$ and $$$r_i$$$ denoting the children of vertex $$$i$$$ ($$$0 \le l_i,r_i \le n$$$).</p><p>For each vertex, $$$l_i=r_i=0$$$ is given if the vertex has no children. Otherwise, $$$l_i$$$ and $$$r_i$$$ are the left and right children of vertex $$$i$$$.</p><p>It is guaranteed that the input defines a valid binary tree satisfying the conditions given in the statement.</p><p>It is guaranteed that the sum of $$$n$$$ over all test cases does not exceed $$$300\,001$$$.</p></div><div class="output-specification"><div class="section-title">Output</div><p>For each test case, output $$$n$$$ integers $$$\tau_1,\tau_2,\ldots,\tau_n$$$ separated by spaces.</p><p>Here, $$$\tau_k$$$ denotes the total time it takes to reach vertex $$$0$$$ if Bob started on vertex
